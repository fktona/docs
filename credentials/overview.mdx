Vela uses two types of credentials, each for a different purpose. Both are generated exclusively in the [Vela dashboard](https://app.vela.dev).

## Credential types

| Credential | Format | Scope | Used for |
|------------|--------|-------|----------|
| **Client secret** | `vela_cs_...` | Account-level | Managing apps, schemas, notification rules |
| **API key** | `vela_live_...` | App-level | Ingesting events |

## How they're used

### Client secret

Used with the **Management Client** to authenticate account-level operations. Passed as a `Bearer` token in the `Authorization` header.

<CodeGroup>

```typescript TypeScript
const client = new VelaManagementClient(process.env.VELA_CLIENT_SECRET);
```

```python Python
client = VelaManagementClient(os.environ["VELA_CLIENT_SECRET"])
```

</CodeGroup>

### API key

Used with the **Ingest Client** to send events. Passed in the `x-api-key` header. Each app has its own API key.

<CodeGroup>

```typescript TypeScript
const ingest = new VelaIngestClient(process.env.VELA_API_KEY);
```

```python Python
ingest = VelaIngestClient(os.environ["VELA_API_KEY"])
```

</CodeGroup>

<Warning>
  Never commit credentials to source control. Use environment variables or your CI/CD secret store.
</Warning>
